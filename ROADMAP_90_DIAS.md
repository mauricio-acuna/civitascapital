# Roadmap 90 Días — Civitas Capital

## Principio rector

Durante 90 días, todo debe servir a una meta:

> Demostrar que Civitas Pro convierte leads inmobiliarios en oportunidades financiables y monetizables.

No se debe ampliar alcance salvo que ayude directamente a vender, medir o demostrar el MVP.

## Días 1-15 — Foco y demo base

Objetivos:

- Cerrar narrativa de producto.
- Preparar datos sintéticos.
- Definir flujo demo.
- Identificar el mínimo backend necesario.
- Eliminar fricción de arranque del repositorio.

Entregables:

- `README.md` investor-ready.
- `PITCH_INVERSORES.md`.
- `MVP_SCOPE.md`.
- `DEMO_SCRIPT.md`.
- `.env.example` por módulo prioritario o global.
- `.gitignore`.
- Checklist de ejecución local.

Criterio de éxito:

- Una persona externa entiende el producto en menos de 3 minutos.
- Una demo puede explicarse sin enseñar todos los módulos.

## Días 16-30 — MVP funcional

Objetivos:

- Crear flujo funcional de cualificación.
- Usar datos sintéticos.
- Reducir dependencia de servicios incompletos.

Entregables:

- Endpoint o script de demo para cualificar comprador.
- Dataset demo de compradores, inmuebles, zonas y productos financieros.
- Resultado de simulación con `VIABLE`, `AJUSTADO` o `NO VIABLE`.
- Resumen de lead cualificado.

Criterio de éxito:

- Se puede ejecutar una demo completa en local sin depender de integraciones externas reales.

## Días 31-45 — Panel o demo visual

Objetivos:

- Hacer visible el valor.
- Preparar experiencia para inversores y agencias.

Entregables:

- Pantalla o prototipo web simple.
- Vista de lead.
- Vista de capacidad financiera.
- Vista de inmuebles compatibles.
- Vista de servicios recomendados.

Criterio de éxito:

- La demo puede hacerse en 7 minutos y muestra claramente dónde se gana dinero.

## Días 46-60 — Pilotos comerciales

Objetivos:

- Validar dolor con clientes reales.
- Conseguir feedback de pricing.
- Ajustar propuesta comercial.

Entregables:

- One-pager comercial.
- Lista de 30 agencias/brokers objetivo.
- Guion de llamada.
- Formulario de feedback.
- Tabla de pilotos.

Criterio de éxito:

- 3-5 conversaciones reales.
- 1-2 pilotos interesados.
- Objeciones documentadas.

## Días 61-75 — Endurecimiento técnico

Objetivos:

- Subir credibilidad técnica.
- Reducir riesgos visibles.

Entregables:

- Build verificado de módulos prioritarios.
- Tests básicos del flujo MVP.
- Documentación de arquitectura resumida.
- Limpieza de artefactos no aptos para GitHub.
- Preparación de CI mínimo.

Criterio de éxito:

- El repositorio no parece experimental desordenado.
- Un técnico puede ejecutar o revisar el MVP.

## Días 76-90 — Paquete inversor

Objetivos:

- Preparar ronda o presentación.
- Convertir el piloto en historia financiera.

Entregables:

- Pitch deck de 10 slides.
- Demo script final.
- Métricas de piloto.
- Pricing preliminar.
- Plan de uso de fondos.
- Roadmap post-inversión.

Criterio de éxito:

- Se puede defender una ronda pre-seed con foco, demo y señales comerciales.

## No hacer durante estos 90 días

- Reescribir todos los microservicios.
- Perseguir integraciones bancarias complejas.
- Crear un portal inmobiliario completo.
- Renombrar todos los paquetes internos si no bloquea la demo.
- Construir funcionalidades que no aparezcan en la demo o en ventas.
